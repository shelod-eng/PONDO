3. THE 5-STEP TRUST-LOCKED PAYMENT JOURNEY
1 Press Buy
Initiates payment journey via PONDO checkout
▸API call fetches customer + product data from partner
eCommerce (Amazon, Temu, Takealot)
▸Embedded video explainer for first-time users
▸Customer redirected from partner site to PONDO secure
checkout
▸Session token issued, cart data encrypted and stored in
Postgres
2 Confirm Details
Identity confirmation & OTP verification
▸Auto-populate: Full Name, SA ID Number, Address, Email,
Geo-location
▸OTP sent via Twilio (SMS) or SendGrid (Email) — 5-min
expiry
▸Mandatory T&C checkbox — transaction cannot proceed
without acceptance
▸POPIA-compliant data capture, AES-256 encrypted at rest
3 Quick Credit & KYC
Third-party identity & affordability checks
▸TransUnion / Experian API: ITC credit check, bureau score
pull
▸KYC: ID scan + face-match verification (third-party
biometric service)
▸Affordability vetting: income vs. purchase cost algorithm
▸Fraud scoring via Python ML microservice (anomaly
detection)
4 Confirm Route
Cost-sharing & final T&C finalisation
▸Define payment split: merchant absorbs X%, customer
pays Y%
▸Display final transaction cost, shipping, and delivery
method
▸Final T&C acceptance signature captured
▸PED (Payment Enabled Delivery) route assigned to KYC-
cleared driver
5 Completed — Transaction Cleared
Fulfillment handoff & PED delivery loop
▸Transaction cleared status posted to Postgres + partner eCommerce webhook
▸PED Device (Yoco/SoftPOS) assigned to vetted courier for field delivery
▸Vetted Custody: KYC-cleared drivers, digital or physical cash handover tracked
▸Centralized Return Protocol: PED device returned to hub at end of shift to close operational loop
▸Customer notification (SMS + email): "Your order is on its way — PONDO Verified"

4. POSTGRESQL DATABASE SCHEMA (SUPABASE)
-- Users / Customers
CREATE TABLE pondo_users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  full_name   TEXT NOT NULL,
  id_number   TEXT UNIQUE NOT NULL,          -- SA ID — encrypted
  email       TEXT UNIQUE NOT NULL,
  phone       TEXT,
  address     TEXT,
  geo_lat     DECIMAL, geo_lon DECIMAL,
  affordability_score DECIMAL DEFAULT 0,
  kyc_status  TEXT DEFAULT 'pending',        -- pending | verified | failed
  itc_status  TEXT DEFAULT 'pending',
  tc_accepted BOOLEAN DEFAULT FALSE,
  created_at  TIMESTAMPTZ DEFAULT now()
);
-- Orders / Transactions
CREATE TABLE pondo_orders (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID REFERENCES pondo_users(id),
  partner_site    TEXT,                      -- amazon | temu | takealot
  product_id      TEXT, product_name TEXT, amount DECIMAL,
  merchant_share  DECIMAL, customer_share DECIMAL,
  step            INT DEFAULT 1,             -- 1-5 journey tracker
  status          TEXT DEFAULT 'initiated',  -- initiated | pending | cleared | failed
  otp_verified    BOOLEAN DEFAULT FALSE,
  fraud_score     DECIMAL DEFAULT 0,
  ped_driver_id   UUID,
  created_at      TIMESTAMPTZ DEFAULT now()
);
-- Audit Log (POPIA compliance)
CREATE TABLE pondo_audit_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID, order_id UUID, action TEXT,
  metadata JSONB, ip_address TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

7. INTEGRATION ARCHITECTURE & DATA FLOW
Customer @ Amazon/Temu/Takealot
        │
        ▼
[ PONDO Checkout Widget / Redirect ]
        │
   Step 1: POST /api/pondo/fetch-cart
        │  → eCommerce Partner API → product + customer details
        │  → Session created in Postgres
        │
   Step 2: POST /api/pondo/send-otp
        │  → Twilio SMS / SendGrid Email → OTP to customer
        │  → Customer accepts T&C checkbox
        │
   Step 3: POST /api/pondo/kyc-check
        │  → TransUnion API → ITC bureau score
        │  → Experian API → affordability vetting
        │  → Python ML Microservice → fraud score
        │
   Step 4: POST /api/pondo/confirm-route
        │  → Cost split calculated (merchant % / customer %)
        │  → PED driver assigned (KYC-cleared)
        │
   Step 5: POST /api/pondo/complete
           → Transaction cleared → Postgres updated
           → Webhook to partner eCommerce (order fulfilled)
           → PED delivery initiated (Yoco/SoftPOS device)
           → SMS + Email: "Verified delivery en route"
